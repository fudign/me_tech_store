# Requirements: Xiaomi Store

**Defined:** 2026-01-22
**Core Value:** Покупатели могут легко найти нужную технику Xiaomi, увидеть все характеристики и цены, и быстро оформить заказ без лишних шагов

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Catalog (Каталог)

- [x] **CAT-01**: Покупатель может просматривать список товаров в категории
- [x] **CAT-02**: Покупатель может просматривать детальную страницу товара с фото и описанием
- [x] **CAT-03**: Покупатель может фильтровать товары по цене
- [x] **CAT-04**: Покупатель может фильтровать товары по характеристикам (память, цвет)
- [x] **CAT-05**: Покупатель может искать товары по названию
- [x] **CAT-06**: Товар отображает все характеристики (память, процессор, камера и т.д.)
- [x] **CAT-07**: Товар показывает несколько фотографий
- [x] **CAT-08**: Товар показывает цену и старую цену (если есть скидка)
- [x] **CAT-09**: Главная страница отображает популярные товары

### Cart & Checkout (Корзина и оформление)

- [x] **CART-01**: Покупатель может добавить товар в корзину
- [x] **CART-02**: Покупатель может изменить количество товара в корзине
- [x] **CART-03**: Покупатель может удалить товар из корзины
- [x] **CART-04**: Покупатель видит общую сумму заказа
- [x] **CART-05**: Покупатель может оформить заказ без регистрации (указав имя и телефон)
- [x] **CART-06**: Покупатель может выбрать способ оплаты (наличными при получении)
- [x] **CART-07**: Покупатель может выбрать онлайн оплату
- [x] **CART-08**: Покупатель может выбрать рассрочку
- [x] **CART-09**: Покупатель получает подтверждение заказа с номером

### Orders (Заказы)

- [x] **ORD-01**: Администратор получает уведомление о новом заказе
- [x] **ORD-02**: Администратор может просматривать список всех заказов
- [x] **ORD-03**: Администратор может просматривать детали заказа
- [x] **ORD-04**: Администратор может изменить статус заказа (Новый → В обработке → Доставляется → Выполнен)
- [x] **ORD-05**: Система сохраняет информацию о покупателе (имя, телефон)
- [x] **ORD-06**: Система сохраняет состав заказа с ценами на момент покупки

### Admin Panel - Products (Админка - Товары)

- [x] **ADM-01**: Администратор может добавить новый товар
- [x] **ADM-02**: Администратор может редактировать существующий товар
- [x] **ADM-03**: Администратор может удалить товар
- [x] **ADM-04**: Администратор может загрузить несколько фотографий товара
- [x] **ADM-05**: Администратор может указать название, описание, цену товара
- [x] **ADM-06**: Администратор может указать характеристики товара (память, цвет и т.д.)
- [x] **ADM-07**: Администратор может указать старую цену (для отображения скидки)
- [x] **ADM-08**: Администратор может назначить товар в одну или несколько категорий

### Admin Panel - Categories (Админка - Категории)

- [x] **ADMC-01**: Администратор может добавить новую категорию
- [x] **ADMC-02**: Администратор может редактировать категорию
- [x] **ADMC-03**: Администратор может удалить категорию
- [x] **ADMC-04**: Администратор может указать название и описание категории

### Admin Panel - Settings (Админка - Настройки)

- [x] **ADMS-01**: Администратор может изменить номер телефона сайта
- [x] **ADMS-02**: Администратор может изменить адрес магазина
- [x] **ADMS-03**: Администратор может изменить email для связи
- [x] **ADMS-04**: Администратор может редактировать информацию в footer

### SEO

- [ ] **SEO-01**: Каждая страница товара имеет уникальные мета-теги (title, description)
- [ ] **SEO-02**: Каждая страница категории имеет уникальные мета-теги
- [ ] **SEO-03**: URL страниц используют ЧПУ формат (/smartfony/xiaomi-14-ultra)
- [ ] **SEO-04**: Сайт генерирует sitemap.xml для поисковых систем
- [ ] **SEO-05**: Сайт имеет файл robots.txt
- [ ] **SEO-06**: Администратор может редактировать мета-теги для товаров
- [ ] **SEO-07**: Администратор может редактировать мета-теги для категорий

### Design & UX (Дизайн и UX)

- [x] **UI-01**: Сайт использует существующий HTML/Tailwind дизайн
- [x] **UI-02**: Сайт корректно отображается на мобильных устройствах
- [x] **UI-03**: Сайт корректно отображается на планшетах
- [x] **UI-04**: Сайт корректно отображается на десктопе

### Security & Authorization (Безопасность и Авторизация)

- [x] **SEC-01**: Администратор может войти в систему (login)
- [x] **SEC-02**: Администратор может выйти из системы (logout)
- [x] **SEC-03**: Админ-панель защищена от неавторизованных пользователей
- [x] **SEC-04**: Все формы валидируются (защита от SQL injection, XSS)
- [x] **SEC-05**: Rate limiting для форм (защита от спама)
- [ ] **SEC-06**: HTTPS для production окружения

### Error Handling (Обработка ошибок)

- [x] **ERR-01**: Страница 404 для несуществующих товаров/страниц
- [x] **ERR-02**: Обработка ошибок сети (если база данных недоступна)
- [x] **ERR-03**: Уведомления пользователю (успех/ошибка операций)

### Performance (Производительность)

- [x] **PERF-01**: Ленивая загрузка изображений (lazy loading)
- [x] **PERF-02**: Пагинация в каталоге (не грузить все товары сразу)
- [x] **PERF-03**: Кэширование данных (LocalStorage/SessionStorage для корзины)
- [x] **PERF-04**: Оптимизация изображений (сжатие, WebP формат)

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Регистрация пользователей | Гостевой checkout достаточен, регистрация снижает конверсию на 63% |
| Личный кабинет покупателя | Не нужен при гостевом checkout, усложняет первую версию |
| Отзывы и рейтинги | Требует модерацию, не критично для запуска |
| Программа лояльности | Преждевременно до получения первых продаж |
| Многоязычность | Только русский язык в v1, другие языки позже |
| Мобильное приложение | Только веб-версия в v1 |
| Telegram бот для клиентов | Не критично для MVP |
| Social login | Противоречит модели гостевого checkout |
| Wishlist (постоянный) | Требует регистрацию или cookies, не критично |
| Live chat | Можно добавить позже, не блокирует продажи |
